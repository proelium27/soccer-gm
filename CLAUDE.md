# soccer-gm

A client-side, BBGM-style soccer management sim (TypeScript, React, Vite, IndexedDB). `SOCCER_GM_SPEC.md` is the full design brief and M0-M6 build order; `DESIGN.md`/`PRODUCT.md` and `docs/superpowers/specs/*` hold detailed feature/design docs. All milestones M0-M6 are done and merged (see "Milestone status" below).

> **This file is a condensed ledger, not a changelog.** Blow-by-blow constant-retune history (dated before/after values, dynasty-audit numbers) lives in git history and the `docs/` design specs — don't re-add it here. Keep entries to *what a feature is, where it lives, and its non-obvious gotchas.*

## Commands

```bash
npm run dev        # Vite dev server
npm run build      # production build
npm test           # vitest run (full suite)
npm run test:watch # vitest watch
npm run typecheck  # tsc --noEmit
npm run cli        # scripts/cli.ts (headless sim harness for audits)
```

Audit scripts live in `scripts/` (e.g. `divisionAudit.ts`, `weakLeaguesAudit.ts`) — run with `tsx`.

- `test/` — vitest. `test/validation/` holds the M1/M4 §8 gate tests.

## Core invariants & gotchas (durable — read before touching sim code)

- **RNG stream order is load-bearing.** The sim is seeded/deterministic; inserting or removing an `rng` draw shifts every downstream roll (different ratings, names, results). When adding per-player traits or post-hoc attribution, derive them from a *separate* seeded stream (`mulberry32`/`hashInts` off pids/match-intrinsics), never from the shared `rng`. This has caused real bugs (AI deficits, shifted generation) multiple times — see any "RNG-stream-order" mention in git history.
- **OVR scale:** 65 ≈ average starter, 70 ≈ good starter, 75 ≈ a team's best, 80-85 ≈ league-wide elite, 90+ ≈ rare outlier. `OVR_WEIGHTS` (`constants.ts`) drives both generation and progression's `computeOvr`.
- **Anti-inflation (fragile equilibrium — several mechanics exist only to hold it):** youth intake anchors to a fixed-at-generation `academyBase`, never to the club's current roster average (the torn-out anchor caused unbounded ratchet). Potential is a *scout estimate* (`estimatePotential`), never a growth input. Every position's expected lifetime rating arc is ~0/slightly negative. `developmentBias` (persistent per-player trait, deterministic from pid) **must taper to exactly 0 by peak age**, or inflation reopens. `growthDamping` scales down only *positive* deltas as ovr climbs. `YOUTH_BASE_OFFSET` offsets the fact that `generatePlayer` doesn't vary quality by age. Any new mechanic that shifts these must be dynasty-audited (`scripts/`).
- **The user's club is unmanaged by design** in headless audits — no AI free agency/renewals/buys touch it, so it rots and produces *all* extreme worst-case roster/points stats. **Exclude `userTid` from tail metrics** in any audit, or report it separately.
- **Match composites are z-normalized within each competition.** A weak league reads as average in its own matches; cross-league surfaces (the Cup) must pool participants into a *shared* baseline (`simThrough`'s `cupMatchData`) or the weak club gets normalized back up to parity.
- **Composite rollup is position-weighted + star-concentrated (`rollupComposites`, `COMPOSITE_STAR_CONCENTRATION`, 2026-07-21).** `attack`/`defense`/`control` no longer flat-average their position group: each is a position-weighted mean (who drives the phase counts most — ST/W for attack, CM/DM for control, CB for defense) *blended toward the group's single best player* by `COMPOSITE_STAR_CONCENTRATION` (0.3), so an elite individual resists being averaged down by weak teammates ("stars carry their phase"; a balanced XI beats a lopsided higher-OVR one). `finishing` (own shot-share weighting) and `keeping` are deliberately left alone. **Pure function of attributes — no shared-`rng` draw**, so RNG stream order is untouched; it only changes composite *values*, hence match results, hence the `touchStats.test.ts` scoreline hash was rebased a second time (intended, not a touch-attribution regression). Dynasty-audit champion churn before raising the concentration (interacts with league z-norm spread).
- **Shot conversion blends team + individual (`SHOOTER_FINISH_WEIGHT`, 2026-07-20, `simMatchDetailed` only).** `resolveShot` takes a `finishAdj` = the finisher's own skill *relative to his team's average* (shooting for shots, heading for corner headers), added to the team finishing composite for `onTargetP`/`saveP` only — **never xG** (xG stays the team-blind "average attacker" baseline, so goals-vs-xG reveals finishing). Centered on the team average, so it *redistributes* a team's goals toward its best finishers without inflating league scoring; a lone elite finisher on a weak side now genuinely stuffs the sheet (the fast composite-only `simMatch`/montecarlo path is untouched → benchmark gates unaffected). Side effect: the season top scorer rises into the mid-30s (m3-top-scorer gate widened 32→36). Rebased the `touchStats.test.ts` scoreline hash once (intended outcome change, not a touch-attribution regression).
- **Determinism boundaries:** cup rounds and decorative touch-attribution (`passes`/`crosses`) each run on their own seeded streams so they can't perturb league match results — league scorelines stay bit-identical to a fixed baseline hash (`touchStats.test.ts`; the baseline was rebased once for the finishing change above).
- **UI conventions:** no emoji in UI (real icons = hand-written inline SVG, see `AwardIcons.tsx`); player-facing prose is casual/second-person, sentence-cased, no em-dashes.

## Milestone status (per `SOCCER_GM_SPEC.md` §7)

All done and merged:
- **M0** — engine port.
- **M1** — players → composites: generation, per-position OVR, composite rollup + league normalization.
- **M2** — season loop, IndexedDB persistence, web worker, scheduling, `simThrough`, minimal UI.
- **M3** — box scores: per-player attribution, `simMatchDetailed`, `SeasonStats`, box-score/leaders pages.
- **M4** — dynasty mechanics: progression (`players/progression.ts`), aging, retirement, youth intake, free agency, contracts, offseason orchestrator (`offseason.ts`).
- **M5** — match texture: cards, fatigue + AI subs, corners, penalties, in-match injuries (multi-matchday recovery), stoppage time, composite re-rolls after subs/reds/injuries. Sub selection blends fatigue with a live `computeMatchRating` estimate (`subPriority`) to pick who comes *off*; a **bench-quality gate** (`worthSub`/`tiredValue`/`benchValue` in `matchSim.ts`, `SUB_FRESHNESS_BONUS`/`SUB_QUALITY_MARGIN`/`SUB_FATIGUE_RELIEF`) decides whether the swap is worth it at all — a fresh replacement may fall short of the tired starter's ovr by an allowance that grows with the starter's energy deficit, so weak benches hold their tireds on (holds back ~1 sub in 10; rebased the `touchStats.test.ts` scoreline hash, intended). **`SUB_QUALITY_MARGIN` was retuned 1→2.5 (2026-07-23)** — at 1 the gate held back ~1 sub in 5, twice the intended rate, which cost the champion enough late-game quality to push the M1 standings gate (`simThrough.test.ts`, champion 78-94 pts) below its floor on main. Measured over 15 seeded seasons: margin 1 → 78.7 champion pts, 2.5 → 80.8, always-sub → 79.8. Note the gate's own 5-season sample is noisy (single seasons swing 70-87), so it sits near its floor by construction — **re-measure over ≥15 seasons before reading a small move in it as a regression.** The gate is also **form-aware** (`SUB_GATE_RATING_INFLUENCE`, `liveRatingFor`): the tired starter's live `computeMatchRating` (vs the 6.0 baseline) shifts `tiredValue` — a starter playing well is harder to justify hooking, one playing poorly easier — reusing the same signal `subPriority` uses for the who-comes-*off* pick. The form term is mean-zero around baseline, so it redistributes *which* subs are held without moving the ~1-in-10 aggregate; rebased the scoreline hash a second time (intended). `MatchPlayer` now carries `ovr` for this. See **Roster & tactics** for the user "more minutes" flag.
- **M6** — depth + finance slice: budgets, scouting, hype, contracts, transfer valuation, two transfer windows, negotiation, extensions/FA signings. (Design detail beyond this summary lives in a Google Doc the user maintains + `docs/finance-design.md`.)

## Feature ledger (post-M6, all shipped)

Each entry: what it is / where it lives / gotchas.

**Roster & tactics**
- **Starting XI** — user drag-drops between XI and bench, persisted as `StoredTeam.starters` (11 pids, `null`=auto). `resolveXI` (`core/lineup/resolveXI.ts`) falls back to `selectXI` auto-pick when invalid. Only the user's club sets `starters`; AI always auto-picks.
- **Formations** — 9 shapes in `FORMATIONS` (`core/lineup/formations.ts`), `StoredTeam.formation` (default `4-3-3`). A real tactical lever: the match sim rolls up composites from the XI each club's formation fields (`teamFormation`/`teamSlots`). AI clubs auto-pick their strongest shape (`chooseBestFormation`, refreshed end of each transfer window via `assignAIFormations`; never the user's). `PitchField.tsx` + `pitchLayout.ts` render per-formation layouts.
- **ROSTER_CAP** (30, `core/constants.ts`) blocks user FA signings/buys when full; youth intake/AI FA unaffected (AI trimmed to 25 each offseason).
- **"More minutes" flag** — user flags bench players via `StoredTeam.moreMinutes` (pids; user-only, `setMoreMinutes` in `core/lineup/moreMinutes.ts`, Roster bench-row toggle). Carried through `simThrough`→`leagueMatchData` onto each bench `MatchPlayer.minutesBoost`; in-match, a flagged player gets a `SUB_MINUTES_BOOST` ovr credit in the sub decision (helps him win the "who comes on" pick and clear the bench-quality gate) plus a dedicated boost-driven sub path in `attemptSub` that subs him for the weakest on-pitch player at his position. A nudge, not a force — the gate still blocks a clearly-worse flagged player. Migrated (backfills `[]`).

**Match data & stats**
- **Box scores** — `PlayerMatchLine`/`SeasonStats`. **Match Rating** (`engine/matchRating.ts`, FotMob-style 0-10, position-weighted, damped by minutes). `minutesPlayed` reconstructed from event clocks. `tackles`/`interceptions` split (`pickInterceptor`). Decorative `passes`/`crosses`/`foulsCommitted` attributed post-sim on a separate rng stream (scoreline-invariant). `avgRating` rolled into `SeasonStats`.
- **Multi-season history** — `Player.stats[]` is append-only (never pruned), so Leaders browse any past season for free. `LeagueStore.seasonHistory` stores each completed season's final table + champion + awards (Standings had no other persisted history — `league.played` is wiped each offseason). Leaders "All Seasons" adds Career vs Single-Season scope.

**Awards & history**
- **End-of-season awards** — `core/awards.ts` (`computeSeasonAwards`, pure/re-runnable). POTY, Golden Boot, Team of the Season. Formulas layer explicit stat weights + an `ovrBonus` (via `ovrDuringSeason`, reads `Player.hist` not present-day ovr) on top of `avgRating`. `/awards` page; Dashboard "Advance" navigates there post-offseason.
- **Club History** — `core/clubHistory.ts` (pure, derives everything from `seasonHistory`, no schema change). `/history`, view any club.

**Squad development**
- **Youth Academy** (user's club only) — `StoredTeam.academyRoster`, flat stipend wages (`academyContractTerms`). Actions in `core/freeAgency.ts` (`signToAcademy`/`promoteFromAcademy`/`releaseAcademyPlayer`). `ensureUserRosterSafety` auto-promotes to `ROSTER_SAFETY_FLOOR` (18) each offseason so the user's unmanaged roster can't drop below 11 and crash the engine. `/academy`, `/incoming-talent` (youth prospects), `/free-agents` (older FAs). AI youth still goes straight to roster.
- **Dynamic academy attraction** — recent within-division finishing rank shifts youth-intake quality (`core/players/academyForm.ts`). **Zero-sum by construction** (one club's + is another's −), so it can't reopen the inflation ratchet.
- **Youth intake position mix** — `generateYouthIntake` draws each prospect's position weighted by `ROSTER_COMPOSITION` (not uniform), so the pipeline matches roster demand (CB/FB/CM 4 slots each get more prospects than DM/AM's 2). Single rng() draw via a precomputed CDF, so RNG-stream *count* is unchanged (only the mapped position moves) — but generation *values* shift, so this is not byte-identical to pre-weighting saves; `generateWorld` is untouched (it never runs youth intake). Note the low-slot positions (DM/AM, 2 each) still structurally leak their *good* surplus into free agency — a club can only house 2, so a prime 3rd gets trimmed — which is why the `/free-agents` list caps per-position on its unfiltered view (see Free agency).
- **Team Rating** — `core/teams/teamRating.ts` (`computeTeamRating`), XI (equal weight) + geometrically-decayed bench, not a flat roster mean. OVR + POT. Shown on Standings (current only) and user's Roster header.

**Free agency**
- **FA transfer hold** (`core/freeAgency.ts`, `faTransferLocked`) — `signFreeAgent` stamps `Player.faSignedSeason` = the season the signing takes effect (offseason signing → `season+1`, mid-season → `season`). While `league.season <= faSignedSeason` the user can't sell him: `inboundOfferCandidates` skips him, and Roster shows "Can't sell yet (just signed)" instead of List-for-Transfer. Kills the buy-a-free-agent-and-flip-him exploit. Optional field, absent on old saves (= unlocked); preserved through progression/`executeTransfer`.
- **Free moves are recorded as transfers** (`FREE_AGENT_TID` = **-1**, `core/transfers/negotiation.ts`) — a free-agent *arrival* is logged as a fee-0 `CompletedTransfer` whose `fromTid` is this sentinel (not a real tid). Load-bearing because club-by-season history is **reconstructed from `league.transfers` alone** (`teamForSeason` in `PlayerProfile.tsx`; also the OVR chart's club colors and champion attribution) — before this, ~19% of all rostered players displayed the *wrong current club*, since an unrecorded free arrival left them attributed to whatever club their last **paid** move landed at. Written by `signFreeAgentAction`, `signToAcademyAction` (academy = a free arrival at the club; promotion academy→senior needs none, same club) and `runAIFreeAgency` via `offseason.ts` step 4. Gotchas: **(a)** departures into free agency are deliberately *not* recorded (would double the log for no display gain — every surface asks "which club in season N", which a release doesn't change); the one lying case, sign-then-cut in the same offseason, is filtered at offseason step 6.47. **(b)** `teamForSeason` must never seed pre-history from the sentinel — a player can sit on a roster for years with no record at all, so seasons before his first record fall back to his present club. **(c)** AI free churn is ~1-2k signings/season, so it's kept out of aggregate feeds (News Feed, per-window transfer lists, Season Preview top transfers) — the user's own free signings still show. **(d)** Any UI rendering a raw from/to tid must go through `clubDisplayName` (`ui/format.ts`) or the sentinel renders as "Unknown"/"Team -1". **(e)** No shared-`rng` draw, so sim results stay bit-identical; no migration (old saves just keep their pre-fix history).
- **AI FA quality poaching** (`runAIFreeAgency`, second pass) — after filling positional shortfalls, each AI club (worst-first) poaches the single best remaining FA that upgrades a position it's already at target depth in (→ composition+1); `trimRosterSurplus` later keeps the best and releases the now-weakest, so the strongest FAs drain out of the pool before the user shops it. Contract length drawn from a **separate seeded stream** (not shared `rng`) → youth generation stays bit-identical.

**AI General Manager** (replaces rule-scripts with emergent evaluation; full brief in the user's "AI GM Philosophy" doc)
- **Evaluation core** — `core/ai/clubContext.ts` (`deriveLeagueContexts`: squad strength/age/depth + league-normalized **ambition**/**frugality** scalars, *derived from state, not stored*) and `evaluate.ts` (`valueToClub` layers positional-need/timeline/affordability on `trueTransferValue`; `perceivedValueToClub` adds scouting-noise scaled by frugality). Normalization is scoped **within each competition**.
- **AI↔AI market** (`core/ai/transferMarket.ts`) — a player's reservation is his value to his current club; he moves to whoever values him enough more to clear it and afford the fee. Guardrails: availability cap, min-surplus, per-window buy/sell caps, depth floor, cash reserve. Runs once per window. **Need-buy path** (`AI_NEED_BUY_*`, `hasPositionalGap` in `evaluate.ts`): the normal `AI_MARKET_MIN_SURPLUS` gate only fires on *bargains*, which left cash-rich clubs banking money while a position went unfilled. When a buyer has a genuine positional gap there (understaffed vs `ROSTER_COMPOSITION`, or a weak startable hole this player upgrades), its required surplus margin drops to `AI_NEED_BUY_MIN_SURPLUS` (0 = pay a fair price, no discount) and it frees up part of its frugality-driven reserve (`AI_NEED_BUY_RESERVE_RELIEF`, never below `AI_MARKET_RESERVE_FRACTION_MIN` → no deficit). Elites stay unbuyable (priceless-star premium is in valuation, untouched); it only routes affordable, already-available squad players to clubs that need them (a move, not a creation → no inflation risk).
- **Inbound offers for the user's players** (`core/transfers/inboundOffers.ts`) — symmetric negotiation (`respondToAsk`), persisted in `LeagueStore.inboundOffers`. `/incoming-offers`. **List for Transfer** (`StoredTeam.transferListed`) lowers a buyer's surplus bar.
- **World player search** (`searchWorldPlayers` in `core/transfers/recommendations.ts`, "Search all players" panel under Recommended Transfers on `/transfers`) — free-form name/filter search over *every* club's roster (no ovr-band/budget/variety cap), ranked by ovr, capped at `PLAYER_SEARCH_LIMIT` (60), empty with no constraint. Each row carries a `forSale` flag mirroring `makeTransferOffer`'s sale gates (loan/rollover-departure/protected-star/depth-floor) so the UI explains an unbuyable player instead of silently no-op'ing the offer; buyable rows reuse the same `NegotiationControls`/`makeOfferAction` as the recommended list. Pure UI+read layer — no new engine gates, so the difficulty levers (depth floor, protected stars, priceless-star pricing) hold unchanged.
- **Contract renewals** (`core/ai/renewals.ts`) — final-year AI players renewed if value ≥ salary × margin (offseason step 0). Sell-at-peak/replace-aging are emergent (no separate mechanic).
- **News Feed** (`/news`) — all league transfer activity + player accomplishments. Accomplishments (`core/newsEvents.ts`, hat-trick/standout/goal-milestones/generational-arrival) detected at sim time into append-only `LeagueStore.newsEvents` (can't be derived retroactively). Timeline merge in `ui/newsFeedTimeline.ts`.
- **Power Rankings history** — snapshots captured every 5th matchday + finale into `LeagueStore.powerRankingHistory` (`core/teams/powerRanking.ts`).

**Finance**
- `budget` is a **cumulative running balance**, never reset to base — light spending compounds. Bounded by `budgetCap(tier, hype)` (`core/finance/budget.ts`): hype-scaled ceiling between `MAX_BUDGET_FLOOR` and `MAX_BUDGET`, ×`financeScale` per country/tier. Banking past the cap destroys the excess (never causes a deficit).
- Wages: cubic in ovr (`seasonSalaryForOvr`, `core/contracts.ts`), charged **up front** at season start (`chargeSeasonStart`); season-end `settleSeasonEnd` handles prize + hype − scouting only. Mid-season acquisitions charge full season salary at acquisition.
- **Capped values + "protected star" not-for-sale gate** — the load-bearing **difficulty lever** (money buys a good squad, not a great one). Two parts: (1) a steep elite premium above `VALUATION_ELITE_THRESHOLD` (76 ≈ the top of a league) in `trueTransferValue` makes the real difference-makers expensive, but every value/asking price is now **clamped at `MAX_TRANSFER_VALUE` ($350M)** so quoted fees stay believable — no more billion-dollar tags. (2) The genuine best players are removed from the market entirely rather than priced into the stratosphere: `core/transfers/protectedStars.ts` `isProtectedStar`/`protectedStarPids` withhold any AI player who was *best-in-world last season* (`ovr ≥ PROTECTED_STAR_OVR` 80, **or** won POTY/Golden Boot/a Team-of-the-Season place) **and** whose club finished **top-`PROTECTED_STAR_TOP_FINISH` (4)** of a **tier-1** league last season. Wired into `makeTransferOffer`/`acceptCounterOffer` (user buys), `recommendedTransfers` (hidden from targets), and `runAITransferMarket` (AI↔AI, via a `protectedPids` set passed by the caller). The gate is **data-driven off a completed-season record**, not the live league, because the offseason summer market runs *before* the `seasonHistory` entry is appended — that caller (`offseason.ts`) builds the record from the freshly-computed `standings`/`awards`/`compsByTid`; regular-season/winter callers use `lastCompletedSeason(league)`. The **user's own club is never gated** (you always control your sales). Note a global player-quality nerf can *not* substitute for gating the market, because match composites z-normalize league-wide quality away (proven — it left title-win rates unchanged).
- **Finance page** (`/finance`) — pure display of budget/hype, scouting slider, settlement breakdown, wage bill, transfer history, league-wide table.

**World / leagues** (competitions-as-data)
- `core/competitions.ts` — `Competition{id,country,tier,name}`, persisted `LeagueStore.competitions`. Teams carry `compId` (not a raw division number). Every "div 0 vs 1" pair is a loop over competitions. Finance fns take a `scale`, looked up via `financeScale`.
- `worldCompetitions()` = **12 competitions / 240 clubs**: England, Spain, Italy, Germany (equal siblings) + **France & Portugal (deliberately weaker + poorer)** via `COUNTRY_STRENGTH_OFFSET`/`COUNTRY_BUDGET_SCALE` (`constants.ts`). Weakness is invisible in a country's own matches (per-competition z-norm) — it bites only in finance (selling leagues) and the Cup (shared baseline).
- `generateWorld()` — one rng pass; England block is byte-identical to the old England-only generation. `createLeagueState` uses it for all new saves; old saves stay at whatever country set they were created with (never re-run generation). `CLUBS` = 240 fictional identities (no real clubs — trademark caution). New-save UI has a country picker (`NewLeague.tsx`); Customize Teams editor is competition-scoped.
- **Per-league nationalities** — `LEAGUE_NATIONALITY_WEIGHTS` (`core/players/nationalities.ts`), real top-flight breakdowns; drawn on each player's isolated identity sub-stream (RNG-safe). Youth intake is country-aware.

**Promotion/relegation & the second division**
- New leagues generate a full tier-2 division per country; 3-up/3-down straight swap each offseason (`core/promotion.ts`, `computeCountrySwaps`/`applyCompetitionSwaps`, per-country). A swapped club's `academyBase` converges gradually (`stepAcademyBaseConvergence`); finance scale switches immediately.
- **`enforceDivision2Ceiling`** (`core/ai/divisionCeiling.ts`) — the load-bearing anti-drift mechanic. Every offseason, any AI-controlled tier-2 player ≥ `DIVISION_2_REFUSAL_OVR_THRESHOLD` (70) is *deterministically* reassigned to a needy tier-1 club **in his own country** (zero fee, releases the receiver's weakest if capped). Runs twice (after promotion/relegation, and after the summer market). This holds "D2 strongest ≤ D1 average" flat across 100-season dynasties — probabilistic approaches did not (relegation doesn't route through any market decision). `wouldRefuseExtension` (`core/ai/breakoutRefusal.ts`, now a flat ovr≥threshold check) is kept for **user-facing** purposes only (Roster "Wants a move to Division 1" badge/Extend-block; surfacing rival stars as buyable). The user's own club is never swept.
- Market/loan **prevention guards**: a tier-2 club never buys/bids-on/is-loaned an over-threshold player (guards placed *after* jitter draws, per RNG-order), so the game stops manufacturing illegal D2 residents and the sweep only cleans up structural cases (relegated squads, homegrown breakouts).

**Continental Cup** (`core/cup/`) — **Swiss league phase + knockout** (all new cups)
- **20-club field**, qualified by league position (not OVR): strong (big-four) leagues send top 4, weak (France/Portugal) leagues send top 2 (`CUP_STRONG/WEAK_LEAGUE_SLOTS`). **Skips season 1** (no prior table → `cup===null`). Cross-cut by a **shared normalization baseline** (`simThrough`'s `cupMatchData` now pools the whole 20-club field) so weak clubs read as genuinely weaker. `LeagueStore.cup`/`cupHistory`, `/cup`.
- **League phase** (`core/cup/leaguePhase.ts`) — one combined table, each club plays `CUP_LEAGUE_PHASE_GAMES` (6) games. The draw (`drawLeaguePhase`) splits the seeded field into `CUP_LEAGUE_PHASE_POTS` (2) strength pots and builds the schedule **directly as 6 edge-disjoint perfect matchings** (guarantees once-per-round; a build-then-color graph may not be 1-factorable), giving each club 3 opponents from each pot, no same-league pairings, an Eulerian home/away split. Runs on its own seeded rng per round (`LEAGUE_PHASE_STREAM` offset) so league scorelines stay bit-identical. Table tiebreak: pts→GD→GF→wins→seed (no head-to-head, Swiss-unreliable).
- **Split** (`splitLeaguePhase`): top `CUP_LP_DIRECT_QF` (4) → straight to QF; next `CUP_LP_PLAYOFF_TEAMS` (8, ranks 5–12) → single-leg **playoff** (5v12…) for the last 4 QF places; rest (13–20) eliminated. `seedKnockoutFromLeaguePhase` seeds the `CUP_KO_SIZE` (8) bracket when the LP completes.
- **Knockout** is format-aware (`isSwissCup`/`koRoundsOf`/`koRoundMatchdays`/`koLegMatchdays`/`koPrizeByRound`/`koFinalRound`): Swiss = QF→SF→Final (3 rounds). **QF/SF are two-legged, home-and-away, decided on aggregate** (`CupState.twoLegged`, true for all newly-built cups; final and playoff always single-leg). Level after regulation (90' single-leg, or aggregate after both legs) → extra time → shootout; LP games can draw. **The two legs are played on separate matchdays** (`CUP_KO_LEG_MATCHDAYS` = `[[29,31],[33,35],[37]]`: QF legs 29/31, SF 33/35, Final 37). Flow (`playKnockoutLeg`, dispatched by `dueCupLeg` → `{round, leg, twoLeg}`): the first-leg matchday plays all first legs (`playFirstLeg`, plain 90') and holds them in `CupState.koLegs` (no prizes, no bracket advance); the second-leg matchday resolves each held tie on aggregate (`resolveTwoLeggedTie` plays leg 2 with the host swapped, sums scorelines, ET/shootout if level), appends the finished `CupTie`, clears `koLegs`. `mergeLines` folds both legs' box scores into one so cup stats/UI see a single tie line; `CupTie.legs` carries the two 90' scorelines for display. Safe to spread across matchdays because cup matches don't carry fatigue/injuries between each other (own per-matchday composites); the home/away swap + doubled sample is what reduces cup-vs-domestic variance (single-leg knockouts between good teams are near coin-flips — measured, stronger club advance rate ~64%→70% at a strong-favorite edge). Each leg runs on its own seeded stream (`hashInts(lid,season,round,30,leg)`), so the league match stream is untouched. **`simThrough` halts before the user's final** (Dashboard warns; the single-leg final's matchday is unchanged at 37). Prizes: participation (LP) + playoff-win + per-KO-round (prize-only, credited when the tie finalizes). Cup stats live on each tie's `BoxScore` (LP matches carry their own `boxScore`), separate from `SeasonStats`; Player Profile Cup tab. The user's Schedule page lists their cup fixtures (LP/playoff/each KO leg) alongside league games.
- **Legacy cups** (pre-Swiss saves, `leaguePhase===null`): old 16-team straight bracket + `CupPlayIn` kept alive only so an in-progress old cup finishes; `migrate.ts` backfills `leaguePhase`/`playoff` to null and `twoLegged` to false (so any cup already in progress finishes single-leg; the next built cup is two-legged). New cups (built each offseason via `buildCupState`) are always Swiss + two-legged. Matchday layout — two-legged Swiss: LP `[3,7,11,15,19,23]`, playoff `27`, KO legs `[[29,31],[33,35],[37]]`; single-leg Swiss (old in-progress saves): KO `[31,34,37]`; legacy bracket: `[8,16,26,34]`.

**Loans** (`core/loans.ts`)
- Fixed 1-3 season move to another roster; flat non-negotiable fee (`computeLoanFee`) + wage handoff (wages key off roster membership, so moving the pid *is* the mechanism). User lists via `/loans` (Accept/Reject only). AI↔AI loans (`runAILoanMarket`) restricted to age ≤ `LOAN_AI_MAX_AGE` (23) and **players outside their club's starting XI** (only buried youngsters). `processLoanReturns` runs early in `simOffseason`. State: `LeagueStore.activeLoans`/`loanListings`/`loanRejections`. No loan-in browsing UI, no early recall (disclosed scope cuts).

**Scouting fog-of-war on potential** (`core/scouting/potentialFog.ts`)
- Fogs **POT only** (OVR/attributes stay exact) as a low-high range, narrowed by scouting spend + ownership tenure, clearing to exact over ~2-3 seasons. `StoredTeam.scoutingObserved` (pid→first-seen season, user only, reconciled at league creation + each offseason). `<PotDisplay>` replaces every raw potential render; AI clubs unaffected (they have their own `perceivedValueToClub` noise). Team-level aggregates stay exact.
- **Scouting spend is locked per season** — `StoredTeam.nextScoutingSpend` (editable only in offseason) becomes the committed `scoutingSpend` at rollover, closing a peek-then-lower exploit.

## Open design decisions (need a user call before coding)

From the 2026-07-11 bug hunt; each has more than one reasonable fix — discuss options first:
1. **Winter-sale wage dodge** — season wages charged once at offseason settlement against the current holder, so selling a high earner in winter dodges his whole season bill. Pro-rate, charge-at-transfer, or accept+document.
2. **Deficit at the 30-man cap** — the "no club runs a deficit" tuning was only proven for the 25-man composition; a stacked full user squad with a low finish can go negative. Retune, cap scouting relative to wages, or allow user deficits + add debt UI.
4. **Mid-season transfer stat attribution** — `SeasonStats` is one running total; Leaders/Roster credit the whole line to the *current* club. Per-stint stats (schema change) or a "joined mid-season" UI marker.

(Item 3, youth intake position distribution, is resolved — `generateYouthIntake` now weights the position draw by `ROSTER_COMPOSITION` instead of uniform, matching intake to actual roster demand. Item 5, Division-2 long-dynasty drift, is resolved — see `enforceDivision2Ceiling` above.)

**Lower-severity leftovers** (no design input needed, not yet done): no general guard against fielding <11 when a roster drops below 11 healthy players (`selectXI` silently skips slots) — partially covered for the user's club by `ensureUserRosterSafety`; AI and other paths unguarded. `pickReplacement` would hand back an outfielder as a "GK replacement" if GKs could ever be subbed/injured/carded (currently unreachable).
